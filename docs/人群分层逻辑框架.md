# 人群分层逻辑框架

## 概述

本文档定义问卷星自动化系统从"问卷解析"到"subagent 任务下发"之间的核心逻辑——人群分层。

一句话概括：将抽象的约束条件转化为具体的统计分组，每个分组对应一个 subagent，各组按比例分配填写份数，最终产出模拟真人分布的答卷数据。

---

## 1. 三层约束模型

系统在解析问卷后、分层前，需要构建三层约束：

```
                       问卷URL
                         │
                         ▼
┌────────────────────────────────────────────────┐
│ L1: AI 自动推理约束  【免费 / 基础功能】         │
│                                                  │
│ 从问卷标题和题目内容中推断目标人群的外在偏向。    │
│ "足球调查" → 男:82% 女:18%                      │
│ "胡同生活" → 城市:北京市                         │
│ "Z世代创业者" → 深圳+95后+创业者                 │
│                                                  │
│ 输出: ai_inferred_constraints.json              │
└────────────────────────┬───────────────────────┘
                         │

┌────────────────────────────────────────────────┐
│ L2: 用户指定约束  【增值服务 / 收费】            │
│                                                  │
│ 用户在 L1 基础上手动叠加约束条件。               │

## 2. 约束维度体系

### 2.1 五大维度

每份问卷可能涉及的约束维度：

| 维度 | 说明 | 信号词示例 |
|------|------|-----------|
| **地理** | 城市/区域/社区 | 深圳市、北京市、老城区、高新区、城中村 |
| **身份** | 职业/角色/社会身份 | 创业者、外卖骑手、全职妈妈、大学生、程序员 |
| **世代** | 年龄段/世代标签 | Z世代、银发族、80后、00后、未成年 |
| **行为** | 特定经历/行为/消费 | "使用过"、"购买过"、"经历搬迁"、"有房贷" |
| **交叉** | 多维度形成交集 | "深圳"+"Z世代"+"创业者" = 极窄人群 |

### 2.2 刚性分级

| 级别 | 定义 | 效果 |
|------|------|------|
| **刚性约束** | 不满足则不该填此问卷 | subagent 100% 遵守，生成答案前先校验 |
| **弹性约束** | 目标群体有此偏向，但不绝对排除 | 控制人群模板中的比例分布 |

### 2.3 约束来源

| 来源 | 检测方式 | 刚性/弹性 |
|------|---------|----------|
| **标题显式出现** | 关键词匹配 + LLM 提取 | 刚性为主 |
| **题目中显式出现** | 问卷题面直接限定范围 | 刚性为主 |
| **Agent 语境推理** | LLM 常识推断 | 弹性为主 |

## 3. 约束覆盖优先级

```
最终人群画像 = L3基线 ∪ (L1推理 ∩ L2用户指定)

具体规则:
  - L2 用户指定的维度 → 刚性覆盖 L1（用户付费了，他说了算）
  - L1 推理的维度 → 弹性覆盖 L3 基线
  - L3 基线 → 兜底所有未约束维度
  - 同一维度 L1+L2 取交集 → 人群收敛
```

### 收敛示例

```
初始人群                        14亿
  ∩ L1推理: Z世代创业者        → 500万
  ∩ L2用户指定: 深圳+上海       → 100万
  ∩ L2用户指定: 月入>2万        → 20万
  ∩ L2用户指定: 本科以上        → 8万

每次 ∩ 操作 = 一次收敛
最终有效人群 ≈ 8万人
```

## 4. 分层算法

### 4.1 完整流程

```
输入: 三层约束 + 总填写份数 N
  │
  ▼
Step A: 确定分层维度 —— 哪些维度要切分？
  │
  ▼
Step B: 确定分层粒度 —— 每个维度切几段？
  │
  ▼
Step C: 交叉生成分组 —— 笛卡尔积
  │
  ▼
Step D: 精简合并 —— 剔除/合并过小组
  │
  ▼
Step E: 分配份数 —— (组占比 × N)
  │
  ▼
输出: subagent 分配表
```

### 4.2 Step A — 确定分层维度

规则：只对"有差异化比例/值"的维度做切分，纯基线维度不切分。

| 约束来源 | 维度 | 是否分层 | 原因 |
|---------|------|---------|------|
| L2 用户指定 | 城市{深圳,上海} | ✅ 必须分层 | 用户付费指定了离散值 |
| L2 用户指定 | 收入>2万 | ❌ 不分层 | 只有单侧阈值，直接过滤 |
| L1 推理 | 性别{男82%,女18%} | ✅ 建议分层 | 有明确比例分布 |
| L1 推理 | 年龄{峰22-30} | ✅ 建议分层 | 有分布差异 |
| L3 基线 | 教育 | ❌ 不分层 | 无差异化约束，自然波动即可 |
| L3 基线 | 婚姻 | ❌ 不分层 | 同上 |

### 4.3 Step B — 确定分层粒度

连续维度离散化规则：

```
离散维度: 保持原值（如: 性别 → [男, 女]）
连续维度: 按 3-5 段切分（如: 年龄 → [18-24, 25-30, 31-35]）
单侧阈值: 不分层，仅过滤（如: 收入>2万 → 全部组内约束）
```

段数控制原则：

```
段数太少 → 分布失真
段数太多 → subagent 数量爆炸、每组样本失去统计意义

默认: 以每组不少于 5 份为准
```

### 4.4 Step C — 交叉生成分组（笛卡尔积）

将分层维度做全交叉：

```
维度: 性别(2) × 城市(2) × 年龄(3) = 12 组

组1: 男 - 深圳 - 18-24
组2: 男 - 深圳 - 25-30
组3: 男 - 深圳 - 31-35
组4: 男 - 上海 - 18-24
组5: 男 - 上海 - 25-30
组6: 男 - 上海 - 31-35
组7: 女 - 深圳 - 18-24
组8: 女 - 深圳 - 25-30
组9: 女 - 深圳 - 31-35
组10: 女 - 上海 - 18-24
组11: 女 - 上海 - 25-30
组12: 女 - 上海 - 31-35
```

### 4.5 Step D — 精简合并

合并规则：

```
合并条件: 该组的预期份数 < min_sample_size (默认 5)
合并方式: 沿最不重要的维度合并相邻组

例如:
  组7 (女-深圳-18-24): 18份 → 保留 ✓
  组10 (女-上海-18-24): 3份  → 与组11合并
  组12 (女-上海-31-35): 2份  → 与组11合并

  结果:
    女-上海-18-35 (合并组): 3+20+2 = 25份
```

### 4.6 Step E — 分配份数

每组份数计算公式：

```
group_quota[i] = round(N × ratio_gender[i] × ratio_city[i] × ratio_age[i])

最后用最大余数法修正，确保 Σgroup_quota = N
```

## 6. L1 约束推理 Prompt

Agent 解析完问卷后，使用以下 Prompt 推理外部约束：

```
你是一个问卷分析专家。请阅读以下问卷内容，识别所有外部约束。

问卷标题: {title}
问卷题目: {questions}

请按以下五个维度逐一分析：

1. 地理约束 — 问卷是否隐含了特定城市/区域/社区？
   信号词: 地名、区域类型（胡同、城中村、高新区...）

2. 身份约束 — 问卷是否限定了特定职业/角色/社会身份？
   信号词: 职业名、角色名（创业者、外卖骑手、全职妈妈...）

3. 世代约束 — 问卷是否限定了年龄段/世代标签？
   信号词: Z世代、银发族、80后、大学生、未成年...

4. 行为约束 — 问卷是否要求填写者有特定经历/行为？
   信号词: "使用过"、"购买过"、"经历过"、"参与过"...

5. 交叉约束 — 上述约束是否形成交集，导致人群更窄？

对每个识别到的约束，标注:
- dimension: 维度名
- value: 约束值
- source: "explicit"(显式) | "inferred"(推理)
- rigidity: "hard"(刚性) | "soft"(弹性)
- reasoning: 推理依据（从哪道题或标题的哪个词判断的）
- distribution: 建议分布范围

注意:
- 一份问卷可能同时有多个约束，全部列出来
- 显式出现在标题中的约束优先级最高
- 不要过度推理——只标注有明确依据的约束
- 外部约束可能是随机且多元的，不限于单一维度
```

             ▼
┌──────────────────────────────┐
│ 2. L1 AI推理约束             │
│    LLM读取标题+题目 → 外部约束 │
│    → ai_inferred_constraints  │

             ▼
┌──────────────────────────────┐
│ 3. L2 用户指定约束 (增值)     │
│    用户手动叠加城市/年龄/收入  │

└────────────┬─────────────────┘
             ▼
┌──────────────────────────────┐
│ 4. L3 基线填充               │
│    未覆盖维度用人口基线兜底   │
│    → 完整人群画像              │

             ▼
┌──────────────────────────────┐

│    A.确定分层维度             │

             ▼

│ 6. Subagent 任务下发          │
│    delegate_task × N          │
│    每个 subagent 一份 persona  │

└────────────┬─────────────────┘
             ▼
┌──────────────────────────────┐
│ 7. 答案汇总 & 统计输出        │
│    单组分布 + 全局交叉分布     │
│    → 模拟真人数据分布报告      │
└──────────────────────────────┘
```
