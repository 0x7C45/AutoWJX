## 1. 总体流程

```
┌───────────────────────────────────────────────────────────────┐
│                        用户操作                                │
│                                                                │
│  ① 输入 URL + 份数                                             │
│       │                                                        │
│       ▼                                                        │
│  ② 系统自动: 解析问卷 → 一级 AI 读取 → 约束推理 → 模板微调     │
│       │                                                        │
│       ▼                                                        │
│  ③ 用户决策:                                                    │
│     ├── [默认] 确认,直接开始                                    │
│     ├── [可选] 叠加 L2 隐形约束 (增值/付费)                     │
│     └── [可选] 调整安全配置                                     │
│       │                                                        │
│       ▼                                                        │
│  ④ 系统自动: 分层 → 生成 subagent 提示词 → 并行填写 → 录入     │
│       │                                                        │
│       ▼                                                        │
│  ⑤ 结果呈现: 统计分布 + 交叉分析                                │
│                                                                │
└───────────────────────────────────────────────────────────────┘
```

**核心设计原则**：用户只需输入 URL 和份数，系统自动完成解析、推理、分层。用户仅在需要干预时（付费增值/调整配置）才介入。

## 2. Step 1: 输入问卷 URL

### 最简模式

```bash
$ autowjx run --url https://www.wjx.cn/vm/xxxx.aspx --count 500
```

仅两个必填参数。其余全部自动。

### 完整参数模式

```bash
$ autowjx run \
    --url https://www.wjx.cn/vm/xxxx.aspx \
    --count 500 \
    --speed conservative \       # 填写速度档位
    --ip-region china \          # IP 地区限制
    --source-mix auto            # 来源渠道分布
```

## 3. Step 2: 自动解析与一级 AI 读取

用户回车后，系统按以下顺序自动执行，无需用户交互：

```
用户输入 URL + 份数
     │
     ▼
┌────────────────────┐
│ 3.1 问卷解析        │  API 路径(wjx-ai-kit) 或 爬虫路径(autollm_wjx)
│   → Question[]      │  提取: q_index, 题型, 标题, 选项, 跳题逻辑
└────────┬───────────┘
         ▼
┌────────────────────┐
│ 3.2 题目抓取到网站  │  如果部署了 Web 控制台,题目自动同步
│   → 后台可查看      │  用户可通过浏览器查看解析结果
└────────┬───────────┘
         ▼
┌────────────────────┐
│ 3.3 一级 AI 读取    │  LLM 通读全部题目 + 对照基础人群模板(L3基线)
│   → 推理外部约束    │  识别标题/题面中的地理、身份、世代、行为约束
│   → 微调模板比例    │  对基线的性别/年龄/收入等维度做出比例调整
└────────┬───────────┘
         ▼
     输出: 调整后的人群模板 JSON
```

### 一级 AI 输入输出

```
输入:
  - questions.json        (解析后的结构化题目)
  - baseline_template     (L3: 中国人口基线分布)

一级 AI 工作:
  1. 通读所有题目，识别约束信号
  2. 判断五个维度: 地理、身份、世代、行为、交叉
  3. 标注刚性/弹性
  4. 对照基线模板，输出微调后的比例和群组

输出:
  - adjusted_template = {
      L1_constraints: {...},
      base_template_adjustments: {...},
      suggested_groups: [...]
    }
```

## 5. Step 4: 用户决策点

系统只在这一处等待用户输入。用户的选项：

| 操作 | 命令 | 说明 |
|------|------|------|
| **确认开始** | `Enter` | 直接进入分层 + subagent 执行 |
| **增值约束** | `l2` | 付费叠加隐形约束，精准缩小人群（跳转 §5） |
| **安全配置** | `config` | 调整填写速度、IP、来源渠道（跳转 §6） |
| **编辑分组** | `edit` | 手动修改分层结果 |
| **增加份数** | `+N` | 在原有基础上追加 N 份 |
| **查看详情** | `detail` | 展开查看每道题的解析结果 |
| **退出** | `q` | 保存当前状态，退出 |

## 6. Step 5: 增值服务 — L2 隐形约束

文档原文：

> 倘若客户购买隐形约束增值服务，则在人群模板 json 文件回传同时添加用户约束选项，回传至 coding 工具后 agent 会根据 skills 撰写工程性可复制性强的 subagent 提示词

### 6.1 购买入口

用户可通过两个渠道获取增值服务：

```
渠道 1: 闲鱼购买兑换码
  用户购买 → 获得激活码 → 在系统中激活 → 解锁 L2 约束功能

渠道 2: 系统内直接购买
  进入 l2 模式 → 系统提示付费 → 支付 → 即时解锁
```

### 6.2 L2 约束叠加流程

```
用户输入 l2
     │
     ▼
┌────────────────────────────────────────────┐
│ ══════ L2 精准约束 (增值服务) ══════        │
│                                              │
│  当前人群: 约50万人 (基于L1)                  │
│  每增加一个约束 → 人群进一步收敛              │
│                                              │
│  可选维度:                                    │
│  [1] 城市/地域    → 当前: 深圳市 (L1)        │
│  [2] 收入水平     → 当前: 未限制              │
│  [3] 教育程度     → 当前: 未限制              │
│  [4] 消费行为     → 当前: 未限制              │
│  [5] 年龄细化     → 当前: 18-35 (L1)         │
│  [6] 自定义维度   → 手动输入                  │
│                                              │
│  选择维度 (可多选,逗号分隔):                  │
│  > 2,3                                       │
│                                              │
│  → 收入: [>5000 / >1万 / >2万 / >5万]       │
│  > >2万                                      │
│    收敛: 50万 → 约25万                       │
│                                              │
│  → 教育: [本科以上 / 硕士以上]               │
│  > 本科以上                                   │
│    收敛: 25万 → 约10万                       │
│                                              │
│  当前共 2 个约束, 人群: 约10万               │
│  [done / 继续添加 / 删除某个约束]            │
│  > done                                      │
└────────────────────────────────────────────┘
     │
     ▼
  L2 约束 + L1 模板 → 合并回传 → 重新分层
```

### 6.3 模板 JSON 回传

L2 约束不是独立存储的，而是合并进 L1 模板后回传：

```
回传前 (L1):                        用户添加 L2:
{                                    {
  "constraints": {                      城市: ["深圳","上海"],
    "gender": {"男": 0.82},             收入: >2万/月,
    "age": {"range": [18,35]}           教育: 本科以上
  },                                  }
  "groups": [8组]
}

              ↓ 合并回传 ↓

回传后 (merged_template):
{
  "constraints": {
    "gender": {"男": 0.82},          ← L1 保留
    "age": {"range": [18,35]},       ← L1 保留
    "city": ["深圳","上海"],          ← L2 新增
    "income": {">20000"},            ← L2 新增
    "education": ["本科以上"]         ← L2 新增
  },
  "groups": [12组 → 精简至9组],     ← 重新分层
  "convergence_log": [
    "L1: 深圳Z世代创业者 → 约50万人",
    "L2: 叠加城市[深圳,上海] → 收敛至约30万",
    "L2: 叠加收入>2万 → 收敛至约15万",
    "L2: 叠加教育>本科 → 收敛至约10万",
    "分层: 4维度 → 12组 → 精简至9组"
  ]
}
```

### 6.4 Agent 根据 skills 撰写 subagent 提示词

回传后的 merged_template 到达 coding 工具，Agent 使用预置的 `subagent-prompt-generator` skill：

```
skill: subagent-prompt-generator

输入:
  - merged_template.json     (L1+L2 合并后)
  - questions.json           (问卷题目)
  - total_count: 500

处理:
  对每个 group:
    1. 提取 persona 画像
    2. 提取 hard_constraints (刚性,必须满足)
    3. 提取 answer_biases (弹性,比例分布)
    4. 注入跳题逻辑
    5. 计算该组 quota
    6. 按固定模板组装 subagent prompt

输出:
  subagent_prompts/
    SA-1.yaml
    SA-2.yaml
    ...
    _manifest.json
```

每个 subagent prompt 格式固定，变量填充自模板 JSON。同一套 skill 服务于不同用户的不同问卷——工程可复制性强。

## 7. Step 6: 安全配置

文档原文：

> 填写间隔：份与份的间隔，题与题的间隔（都可自定义）
> IP 与来源：是否有特殊锁定 ip 要求（如仅限制中国），线上来源（小程序，链接，手机等）

### 7.1 填写间隔

两个维度：

```
份与份的间隔（宏观）
  → 上一份提交完成 → 等待 N 秒 → 开始下一份
  → 模拟: 一个人填完 → 下一个人打开链接

题与题的间隔（微观）
  → 第1题选完 → 等待 N 秒 → 第2题
  → 模拟: 阅读 + 思考 + 点击选项
```

三个预设档位：

| 档位 | 份间隔 | 题间隔 | 适用场景 | 风险 |
|------|--------|--------|---------|------|
| **conservative** (默认) | 3-8s + 每5份额外5-12s | 简单0.5-2s, 矩阵2-5s, 填空3-8s | 正式调研 | 🟢 低 |
| **standard** | 1.5-4s | 0.3-4s | 一般测试 | 🟡 中 |
| **aggressive** | 0.5-1.5s | 0.1-1s | 压力测试 | 🔴 高 |

自动校准行为：
- 凌晨 0-6 点自动降速 ×1.4
- 每 5 份自动插入批次休息
- 填空/矩阵题自动延长间隔

### 7.2 IP 控制

```bash
# 仅中国 IP
--ip-region china

# 指定省份
--ip-region beijing

# 每 N 份轮换
--ip-rotate 10

# 自定义代理列表
--proxy-file proxy.txt
```

### 7.3 来源渠道模拟

问卷星跟踪答卷来源渠道。系统模拟不同来源的行为差异：

```
来源             问卷星识别        行为特征
────             ──────────        ────────
微信小程序       UA + sojumpparm   停留短、题间隔快
直接链接         Referer=直接      停留中等
手机浏览器       UA=Mobile         题间隔偏快
PC 浏览器        UA=Desktop        题间隔偏慢、填写更完整
```

配置示例：

```bash
# 自动分布
--source-mix auto

# 自定义比例
--source-mix "wechat:40,link:35,mobile:20,pc:5"
```

## 8. Step 7: 人群分层与 subagent 下发

用户确认后，系统自动执行分层引擎（详见 [人群分层逻辑框架.md](./人群分层逻辑框架.md)）：

```
分层引擎运行
     │
     ├── Step A: 确定分层维度 (有差异的维度才切分)
     ├── Step B: 确定分层粒度 (连续维度离散化 3-5 段)
     ├── Step C: 笛卡尔积交叉生成组
     ├── Step D: 精简合并 (预期份数 <5 的组)
     └── Step E: 分配份数 (最大余数法修正)
     │
     ▼
  subagent 分配表 → skill 生成 prompt → delegate_task × N
```

### Subagent 任务包示例

```yaml
# SA-2: 男-深圳-25-30, 82份

subagent_id: SA-2
quota: 82

persona:
  gender: 男
  city: 深圳
  age_range: [25, 30]
  identity: 创业者
  income: 2-5万/月

hard_constraints:
  Q1: [1]              # 性别必须选男
  Q2: [2, 3]           # 出生必须在 Z 世代范围
  Q5: [1, 2]           # 城市必须深圳或上海

answer_biases:
  Q4_brand:
    星巴克: 0.28, Manner: 0.22, 瑞幸: 0.18
    M_Stand: 0.15, Seesaw: 0.10, 其他: 0.07
  Q6_frequency:
    每天: 0.18, 每周3-5次: 0.40, 每周1-2次: 0.28, 偶尔: 0.14

skip_logic:
  - Q2选1 → 终止填写
  - Q2选4 → 终止填写

fill_parameters:
  form_interval: [3, 8]秒
  question_interval_simple: [0.5, 2]秒
  ip_region: china
  source: ["wechat:0.35", "link:0.40", "mobile:0.25"]
```

## 11. CLI 命令参考

### 主命令

```bash
# 最简模式 — 仅 URL + 份数
autowjx run --url <url> --count <n>

# 带安全配置
autowjx run --url <url> --count <n> \
    --speed conservative \
    --ip-region china \
    --source-mix auto

# 跳过交互确认，直接执行
autowjx run --url <url> --count <n> --yes

# 使用已有模板文件 (跳过 L1 推理)
autowjx run --url <url> --count <n> --template template.json
```

### 子命令

```bash
# 仅解析问卷，不填写
autowjx parse --url <url>

# 仅推理约束，不填写
autowjx analyze --url <url>

# 仅分层预览，不填写
autowjx stratify --url <url> --count <n>

# 激活兑换码
autowjx activate --code <redemption-code>

# 查看历史报告
autowjx history
autowjx report --id <task-id>
```

### 完整参数表

```
--url            问卷星 URL (必填)
--count          总填写份数 (必填)
--speed          填写速度: conservative|standard|aggressive (默认 conservative)
--ip-region      IP 地区: china|beijing|shanghai|... (默认 china)
--ip-rotate      每 N 份换 IP (默认不轮换)
--proxy-file     代理列表文件路径
--source-mix     来源分布: auto|wechat:N,link:N,mobile:N,pc:N
--form-interval  份间隔范围(秒): min-max (覆盖 speed 预设)
--question-interval 题间隔范围(秒): min-max
--template       使用已有模板 JSON (跳过 L1 推理)
--output         输出目录 (默认 ./output)
--yes           跳过所有交互确认
```

## 12. 数据流总览

```
问卷URL
  │
  ▼
┌─────────────────────────────────────────┐
│ Step 1-2: 解析 + 一级 AI 读取             │
│   URL → parse → Question[]               │
│   Question[] → LLM → L1 约束推理          │
│   L1 约束 + L3 基线 → 微调模板            │
│   → adjusted_template.json               │
└──────────────┬──────────────────────────┘
               ▼
┌──────────────────────────────┐
│ Step 3: 用户决策              │
│   确认[Enter] / L2增值 / 配置  │
└──────────────┬───────────────┘
               │
     ┌─────────┼──────────┐
     ▼         ▼          ▼
  [Enter]    [L2]     [config]
  直接进入   叠加约束   调整安全参数
     │         │          │
     │         ▼          │
     │   付费 → L2约束     │
     │   L1+L2合并         │
     │   ↓                │
     │   merged_template   │
     │         │          │
     └─────────┼──────────┘
               ▼
┌─────────────────────────────────────────┐
│ Step 4: 人群分层                          │
│   约束 → 分层引擎(A-E) → subagent分配表   │
│   → subagent-prompt-generator skill     │
│   → subagent_prompts/SA-*.yaml          │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│ Step 5: Subagent 执行                    │
│   delegate_task × N                     │
│   每个 subagent: persona + quota + bias  │
│   → submitdata 字符串                   │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│ Step 6: 自动录入                          │
│   API: submit_response() 或              │
│   脚本: HTTP POST 模拟提交               │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│ Step 7: 结果输出                          │
│   单组分布 + 全局交叉分布                 │
│   → report.json + responses.csv         │
└─────────────────────────────────────────┘
```
